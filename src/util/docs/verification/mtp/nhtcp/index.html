1.Introduction

1.1) Purpose and Scope

This document provides a test plan for generic network handler module which is the part of basic utility kit. It has two following interfaces.

  1. Management Interface .
  2. User (Appl) Interface.

1.2) References

[mdd] : Module Design Document of TCP Network Handler.
[C++NP] : C++ Network Programming : Douglas C. Schmidt and Stephen D. Huston .

1.3) Definitions, Acronyms and Abbreviations

  1. ASM : Application System Management.
  2. ACK : Acknowledgement.
  3. FSM : Finite State Maching.
  4. MI : Management Interface.
  5. UI : User Layer Interface.
  6. TCP : Transport Control Protocol.
  7. V : Valid Message or Valid Parameter to function
  8. I : Invalid Message or Invalid Parameter to function
  9. R : Remote test stub function call.
  10. L : Local test stub function call.
  11. VREO : Valid parameter and Remote end opened
  12. VREC : Valid parameter and Remote end closed
  13. BP : Bound Port(Port in use)
  14. CL : Communication Lost
  15. RA : Remote Application
  16. IUT : Implementation Under Test

  • User Layer Interface Functionalities.

    • 3. Features to be tested

    1. Management Interface Functionalities.
      1. Init Network Handler.
      2. Create Network Handler.
      3. Initial Configuration of Network Handler.
      4. General Configuration of Network Handler.
      5. Start Network Handler.
      6. Check the Status of Network Handler.
      7. Check the Statistics Report at any point of time.
      8. Network Handler Creation without initial configuration.
      9. Network Handler Creation more than once.
      10. Starting Network Handler without creation.
      11. Send Message without Starting the Network Handler.
      12. Try Initial Configuration more than once.

    2. User Layer Functionalities
      1. Opening and Closing TCP Connection.
      2. Opening and Closing TCP Listener.
      3. Sending and receiving of Messages.

      Note: User Layer Functionalities Testing should be done with the various conditions,

      1. With a single tcp connection, run test case for send/receive.
      2. Mulitple tcp connection (1000), run test case for send/receive.

    4. Features not to be tested


    5. Approach

    The testing approach is different as per MI and UI interfaces. MI testing functionalities is based on state transitions of TCP network handler and System Management API parameter validation. UI testing functionalities is based on the normal tcp connection, listeners states. and also User Layer API parameter validations.

    5.1 Description

    Black box testing approach is used which just checks what a program is supposed to do without going into the program structure.

    None

    Management Interface Functionalities

    Item Information
    Test Case TCP/MI/IDLE/INIT_NH(V)
    Purpose To test init network handler function.
    Pre-Test Condition Null.
    Expected Outcome MI will get CdSuNhTcpInstId.
    Test Description Call function cdSuNhTcpMiInit with valid parmeters such as.
    1. Valid SM Actor Id.
    2. Valid NH Actor Id.
    3. Valid range of Maximum no. of connections.
    4. Valid range of Maximum no of Lstns.
    5. Valid range of Maximum buffer length.
    6. Valid range of Time to wait in the select call waiting for connections.(microsecs)
    7. Valid range of Time to wait in the select call waiting for data in connection.(microsecs)
    8. Valid range of Time to sleep if no fds are there.
    9. Valid range of Maximum length of the queue of Listen Queue.
    10. Layer manager will receive Valid CdSuNhInsId.

    Item Information
    Test Case TCP/MI/IDLE/INIT_NH(I)
    Purpose To test init network handler function.
    Pre-Test Condition Null.
    Expected Outcome MI will get CDSU_NHTCP_INV_NHINSTID as a CdSuNhTcpInstId.
    Test Description Call function cdSuNhTcpMiInit with either of these invalid parmeters such as.
    1. Invalid SM Actor Id.
    2. Invalid NH Actor Id.
    3. Invalid Maximum no. of connections.
    4. Invalid Maximum no of Lstns.
    5. Invalid Maximum buffer length.
    6. Invalid Time to wait in the select call waiting for connections.(microsecs)
    7. Invalid Time to wait in the select call waiting for data in connection.(microsecs)
    8. Invalid Time to sleep if no fds are there.
    9. Invalid Maximum length of the queue of Listen Queue.
    10. Layer manager will receive Invalid CdSuNhInsId.
    Any other API call before the cdSuNhTcpMiInit().
