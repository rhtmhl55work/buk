1.Introduction

1.1) Purpose and Scope

This document provides a test plan for generic network handler module which is the part of basic utility kit. It has two following interfaces.

  1. Management Interface .
  2. User (Appl) Interface.

  2. UI : User Layer Interface.

    3. Features to be tested

    Features to be tested are given bellow. Each category has to be tested for default and non default eventhandler. In case of default eventhandler response or indication comes through message.

    1. Management Interface Functionalities of UDP NH Instance.
      1. Init Network Handler.
      2. Initial Configuration of Network Handler.
      3. General Configuration of Network Handler.
      4. Start Network Handler.
      5. Stop Network Handler.
      6. Check the Status of Network Handler.
      7. Get the Statistics Report.

    2. User Layer Functionalities of UDP NH Instance
      1. Local Transport Address Only Fixed and Local Port is Ephemeral port.
        1. Adding Udp Port.
        2. Sending Data.
        3. Data Indication.
      2. Local Transport Address Only Fixed and Local Port is Standard port.
        1. Adding Udp Port.
        2. Sending Data.
        3. Data Indication.
      3. Destination Transport Address Only Fixed.
        1. Adding Udp Port.
        2. Sending Data.
      4. Both Local And Destination Both Transport Address Fixedand local port no is ephemeral.
        1. Adding Udp Port.
        2. Sending Data.
        3. Data Indication.
      5. Both Local And Destination Both Transport Address Fixed and local port no is standard.
        1. Adding Udp Port.
        2. Sending Data.
        3. Data Indication.
      6. Both Local And Destination Both Transport Address Not Fixed.
        1. Adding Udp Port.
        2. Sending Data.
      7. Removing Udp Port.

    3. Default Event Handler Test

      All the test cases related to Management interface and User Interfaces are valid test cases here. And the support of multiple user of one UDP NH should be tested(Since Message Interface it is easy to test multiple user support).

    4. Invalid Sequence Of Operation

      When System Manager or User does out of sequence of operation UDP Nh should behave properly. This feature has to be tested.

    5. Configure Parameter Test
      1. Maximum no of UDP Port.
      2. Maximum size of Buffer.
      3. Data Select Timer.

    6. Multiple UDP NH Instance Support

      This utility must support existance of multiple UDP NH Instances.

    7. Multiple User Support

      This utility must support existance of multiple user for one UDP NH Instances.

    8. Multiple UDP NH Instance and Multiple User Support

      This utility must support existance of multiple UDP NH Instances and multiple User.

    9. Staticsics Package Test

      Do Various Type of operation on UDP NH after each operation counter should be updated properly.

    10. Miscellaneous Test Cases

    11. Load Testing
      1. Open maximum no of udp port possible.
      2. Open x NH instance y udp port send data over each udp port. (Each isntace has y/x udp port). In remote end open one receiver. then do data transfer.

    4. Features not to be tested


    5. Approach

    The testing approach is different as per MI and UI interfaces. MI testing functionalities is based on state transitions of UDP network handler and System Management API parameter validation. UI testing functionalities is based on the udp port, type of udp port, and also User Layer API parameter validations.

    5.1 Description

    Black box testing approach is used which just checks what a program is supposed to do without going into the program structure.
