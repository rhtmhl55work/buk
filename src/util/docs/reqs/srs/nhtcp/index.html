1. Introduction

1.1 Purpose

The purpose of this document is to capture the functional, performance, operational and interface requirements of a generic TCP Network Handler for the Internet Telephony System. The module needs to be build such that it can be used with all stacks like H.323, SIP, Meagaco or any other module.

1.2 Scope

The reader is assumed to be familiar with the TCP related and the document in the references. Here only the specifications are given. The Architecture and detail document will be available later.

1.3 Definitions, Acronyms and Abbreviations

Abbreviations

API : Application Programming Interface
ACK : Acknowledgement
MI : Management Interface
UI : User Layer Interface
TCP : Transport Control Protocol

TBD : To Be Done

Definitions

Connection A full association that is used by peers to exchange data between endpoints of a network application.
Listener Listenes on a particular port for connections.
TCP Network Handler The TCP Netwok Handler offers the basic tcp network services like open connection, send data on connections, receive data from connections, set listeners, stop listeners and close connection etc.
TCP Network Handler Instance Id It is the unique identity of newly created TCP Network Handler.
System Management System Management is responsible for creating, configuring, deleting of TCP Network Handler module.
User Layer User Layer is the customer of TCP Network Handler which avails the services offered by it.

2. General Description

2.1 Module Perspective

2.1.1 Background

Most of the modules in the internet telephony system uses network services to interact with each other. In a case of IP-IP call, we need to establish the connections between end points and to communicate on established connections. The lowest essential block in such modules provides the transport mechanism. It offers all network services which upper layers requires to communicate across.

2.1.2 Overview of Network Handler

The main purpose of the tcp network handler is to transport packets over IP network by managing multiple connections. It needs to offer the general services like

  • Opening TCP Connections.
  • Closing TCP Connections.
  • Start TCP Listeners.
  • Stop TCP Listeners.
  • Send data on TCP connection.
  • Receive data on TCP connection.

2.1.3 Features of Network Handler

Features Support

It should support the following feature.

  1. Reliable communication.
  2. Offering generic services.
  3. High level of abstraction.
  4. Management of multiple connections.
  5. Management of multiple listeners.
  6. Statistics Report.
  7. Supports TPKT.

2.1.4 Event Handler

Both System Management and User Layer should be registerd with the event handler (as call back function).

The System Management will register the event handler with Network Handler. The responses for MI API function calls and events will be notified by Netwok Handler to the registered event handler.

Similarly, the User Layer will register with the two event handler. One is for listener events and another is for connection events. The responses for UI API function calls and events of listener and connection will be notified by Network Handler to the registered listener and connection event handler respectively.

2.1.5 API

Network handler should have function based API for the follwoing interfaces.

  1. System Management Interface
  2. User Layer Interface
All the API calls will take essentials parameter as argument structure to perform intended operation.

The intended functionality of the APIs is given below:

1. Management API
  • Configuration of Network Handler
    API calls for Initial (static) and General (dynamic)configuration.

  • Status of Network Handler
    The NH should report its status like RUNNING, STOPPED, CREAETED.

  • Controling of Network Handler
    The control functions like
    - Create NH.
    - Delete NH.
    - Start NH.
    - Stop NH.

  • Statistics of Network Handler
    The NH should report with the following statistics.
    - Total number of Add Listener Requests.
    - Total number of Connection Requests.
    - Total number of Data Requests.
    - Total number of Disconect Connection Requests.
    - Total number of Remove Listener Requests.
    - Total number of Connection Indications.
    - Total number of Data Indications.
    - Total number of Close Connection Indications.
    - Total number of Successful Connection Requests.
    - Total number of Successful Add Listeners.
    - Total number of Successful Data Transmits.
    - Total number of Successful Close Connections.
    - Total number of Successful Remove Listeners.
    - Total number of Current Active Connections.
    - Total number of Current Active Listeners.
2. User Layer Interface API
  • Opening TCP Connections.
    It will take remote IP address and port as parameters and returns with the connection id or handle and response code.

  • Closing TCP Connections.
    It will take connection id/handle as parameter and returns with response code.

  • Start TCP Listeners.
    It will take self IP addresss and port as parameter and returns with the listener id or handle and response code.

  • Stop TCP Listeners.
    It will take listener id/handler as parameter and returns with response code.

  • Send data on TCP connection.
    It will connection id /handler and data buffer as parameters returns with response code.


2.1.6 Multi-Network Handler Support.

The overview of Network Handler depicts the basic functionality expected by the upper layer. The idea of providing multiple instances of such network handlers to upper layer leads to have Multi-Network Handler support. The upper layer can create as many instances of network handler as per the requirement. These requirement can arise in certain cases like in load conditions, to have separate network handlers for self opened connections and remotely opened onnections. This gives the feature of load sharing. The network handler instance manager should be singleton modules in the system.

2.2 Module Functions

2.2.1 Module Initialization
  • NH Module Init : This will return the NH instance id.
  • Configure NH : Two Types of configuration Initial and General. Inital configuration parameters are like maximum connections, maximum listener, size of queues etc. General configuration parameter can be used for dynamic configuration of NH.
  • Create NH : This is control call to create NH.
  • Start NH : It will make NH in ready state.

2.3 User Characteristics

As there are two interface, System Manager and User Layer. Both the interface registers with event handlers as a call back function to get responses from NH on appropriate API calls. In User Layer, it will have two call back function, one is for listeners and another is for connections. So whenever any events occurs on listener, NH will call the listener event handler as a call back and similarly any events on a connection, NH will call connection event handler.

2.4 Constraints

The id (handle) for connections or listener will be allocated by NH not by the user layer.

3.2 Functional Requirements

  • Support for Multiple Network Handler Creation.
  • Support all network services like
    • send data.
    • receive data.

3.3 Performance Requirements

  • It should have high throughput in Load Conditions.
  • It should have low run-time memory.
  • It should have efficient time management and memory management.
  • It should be thread safe and re-entrant.
